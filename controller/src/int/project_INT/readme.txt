虚拟机测试拓扑入图片所示，总共五个交换机和四台终端，s1和s4扮演的角色就是我们自己的p4交换机，中间的s2，s3，s5模拟的是单纯负责srv6转发的交换机，终端h3和h4负责转发普通数据包，h1和h2负责是INT控制器。

链路1：s1,s2,s3,s4
链路2：s1,s2,s5,s3,s4

receive_4.py（接收从h1到h2走链路1的探测包），receive_5.py（接收从h1到h2走链路2的探测包），receive_7.py（接收从h1到h1走链路1的探测包），receive_normal.py（接收普通数据包），send_INT_4.py(发送h1到h2走链路1的探测包)，send_INT_5.py(发送h1到h2走链路2的探测包)，send_INT_7.py(发送h1到h1走链路1的探测包)，send_normal80.py(发送h1到h2走链路1的普通数据包)，send_normal100.py(发送h1到h2走链路2的普通数据包)，




在虚拟机上进行的测试
测试1：打开终端h3和h4，h3发出普通数据包（源ipv4地址是h3，目的ipv4地址是h4，dport=80），s1在接收到这个数据包的时候会按照这个三元组的匹配规则给它一个服务等级，然后相应地会给这个数据包规定传播的路径（s1，s2，s3，s4），h4可以顺利接收到这个普通数据包

测试2：打开终端h3和h4，h3发出普通数据包（源ipv4地址是h3，目的ipv4地址是h4，dport=100），s1在接收到这个数据包的时候会按照这个三元组的匹配规则给它一个服务等级，然后相应地会给这个数据包规定传播的路径（s1，s2，s5，s3，s4），h4可以顺利接收到这个普通数据包

测试3：当h3只发送（源ipv4地址是h3，目的ipv4地址是h4，dport=80）这个数据包的时候，h1向h2发送单向探测包（单向，s1，s2,s3.s4），可以在h2接收端接收到这天链路上的相关信息，当h3只发送（源ipv4地址是h3，目的ipv4地址是h4，dport=100）这个数据包的时候，h1向h2发送单向探测包（单向，s1，s2,s3.s4），h2的不会收集到相关信息

测试4：h3同时发送两种数据包的信息，同时h1也发送两种INT单向探测包，在h2接收端可以收到得到两条链路的相关信息，并且可以区分出哪个是链路1哪个是链路2

测试5：h1发送双向探测包（链路1的正向和反向），同时h3向h4发送普通数据包(链路1)，h1接收到的INT包只会显示正向的链路信息，反向的相关信息是0
